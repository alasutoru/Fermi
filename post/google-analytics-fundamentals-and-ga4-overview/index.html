GA分析快速學會數據報表解讀

Google Analytics GA 網站分析 utm 參數 資源 資料檢視 GA4 User ID 跨裝置追蹤 參與度
September 9, 24240 · 1 min · 182 words · Fermi Logan | Suggest Changes

Google Analytics (GA) 是目前主流且對媒體友善的網站追蹤工具，其運作原理建立在 utm 參數，並以帳戶 資源 資料檢視三層結構管理資料；最新版本 GA4 則以使用者為核心，整合 Web 與 App 追蹤，並透過 User ID 實現跨裝置分析，建議網站同時安裝通用 GA 與 GA4 以進行多角度數據解讀。

  • 線上流量無形，需要使用分析工具才能得知網站經營狀況。使用工具並有數字行銷概念，才能有效經營並看到流量與業績成長。
  • 網站追蹤分析與監測在數位行銷領域與媒體操作同等重要，是許多行銷人員頭痛的地方。
  • 影片以目前主流、對媒體最友善、被稱為 Google Ads 同捆包的 Google Analytics (GA) 教學媒體採購和行銷企劃所需的評估分析概念。
    • 介紹以行銷人員需求為主，只求速成不求甚解。
  • GA 是一個工具，懂 GA 不代表懂網站分析。其他工具如 SimilarWeb, Mixpanel, Alexa 也各有追蹤強項。網站分析最重要是懂得如何用數據做決策。
  • GA 的運作原理：GA 是 Google 在 2005 年收購 Urchin 公司而來，其技術優勢包括發明 utm 參數 (urchin tracking model)。Google 收購後將 GA 變為免費服務，只要有 Gmail 即可使用。
  • GA 的權限分成管理者和工作人員。
    • 管理者：網站資產擁有者，可新增/移除工作人員、開立 GA 資源追蹤編號、進行幾乎所有變更。
    • 工作人員：三種權限 - 編輯 (可新增追蹤編號、資料檢視、自訂維度指標、連結 Google Ads/Search Console 等)、協作、檢視 (只能使用/查看報表功能，無法擴充分析範圍)。連結 Google Ads 需要 GA 的編輯權限。
  • 通用 GA 的管理介面結構：帳戶 (Account)、資源 (Property)、資料檢視 (View)。
    • 帳戶：名下的 GA 帳號數量 (用多少 Gmail 開的帳號)。
    • 資源 (Property)：GA 的靈魂。包含 GA 追蹤編號 (GATC)、資料收集方式、資料保留期等重要設定。建議一個網站或主網域使用一個 GA 資源編號。
    • 資料檢視 (View)：從資源收集輸出報表的資料。每開一個資源預設建立一個「所有網站資料」View。有資源編輯權限可開新的 View 並設定篩選器。建議保留預設的「所有網站資料」View，因為 View 的數據收集是從創建完成後才開始，無法追溯過去數據，且容易因編輯篩選器而影響數據。
  • GA4 (Google Analytics for Property)：Google 在 2019 年推出的新產品，是最新版的 GA，功能持續更新。
    • GA4 整合了 Google Analytics Web + App 和 Google Analytics + Firebase，重點是跨資產追蹤 Web 和 App。
    • 應用程式追蹤基礎單位是事件 (沒有網頁)，網站追蹤基礎單位是網頁。兩者看的指標不同。
    • Google 希望像 Facebook 一樣做跨裝置使用者分析。Facebook 透過註冊會員達成。
    • GA4 如何做到跨裝置分析：1. 整合所有資產；2. 通過 User ID。User ID 在通用 GA 已有。在 GA4，可將擁有的 App (iOS/Android) 和網站登錄到資料串流，自定義共同的「鍵值」（如會員登錄 ID、手機 IDFA/AAID）作為統一 User ID。不建議使用 IDFA 等隨時可能被拒絕追蹤的資料。
    • 新版 GA4 最重要是以使用者為核心出發，整合網站和應用程式分析指標。
    • GA4 重新整合了工作階段概念，工作階段在 GA4 沒有使用者那麼重要。
    • GA4 將跳出率改為參與度，希望分析者關注有互動的工作階段或使用者。
    • 通用 GA 以網站為核心，GA4 角度非常不同。
  • 學習 GA4：可參考官方示範帳戶、說明中心或業界課程。
  • 即使只有 Web，也建議先升級一個 GA4 帳戶，可以同時用通用 GA 和 GA4 角度解讀資料（一個從網站，一個從使用者）。
  • GA4 的 GATC 需要另外生成並埋設在網站中才能開始收集資料，不會自動抓取。
