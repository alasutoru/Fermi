Facebook 企業管理平台是統整廣告資產所有權、管理內部和合作夥伴人員權限、並進行重要網域認證的中央控制系統。

Facebook 企業管理平台 (Business Manager, BM) 的重要性、功能和權限管理#

  • Facebook 廣告業務在台灣發展多年，系統不斷進化。除了粉絲專頁，還有廣告管理員、Pixel、商品目錄等行銷資產。
  • Facebook 企業管理平台 (BM) 於 2015 年推出，用於管理多個 Facebook 行銷資產 (粉絲專頁、廣告帳號、IG 帳戶等)。它是一個資產管理中心。
  • BM 擁有者能掌握平台內建立資產的所有權，並管理內部人員的資產使用權限，可依團隊或客戶分權。
  • 合作夥伴關係：BM 之間可以建立合作夥伴關係，互相指派資產使用權。常見用於客戶將粉絲專頁指派給代理商代操廣告。這增加了資產運用的彈性。
  • 常見問題：客戶收不到粉絲專頁授權邀請。原因可能包含：
    • 客戶的粉絲團已被另一層 BM 擁有 (客戶可能不知道自己有 BM)。Facebook 過去曾鼓勵 BM 擁有者認領沒有 BM 的粉絲團。授權邀請會傳送到擁有粉絲團的 BM。
    • 客戶曾直接從粉絲專頁按下「推廣」(前台下廣告)。Facebook 會自動為客戶開一個廣告管理員帳號和一個 BM 來管理這些資產。BM 登入帳密與粉絲團小編相同，但入口不同。
  • Facebook 有時會隱密地為使用者創建新的工具或帳號 (如商務套件)。

BM 內部權限分類:#

  1. 相關人員 (People)：管理公司內部人員。
    • 員工 (Employee)：操作 BM 資產。
    • 管理員 (Admin)：操作資產、管理及指派其他資產、新增員工 (加權限)。通常是團隊負責人。
    • 財務 (Finance)：查看/編輯帳單付款資訊，無廣告操作權限。
  2. 合作夥伴 (Partners)：管理 BM 與 BM 之間的合作關係。用於跨 BM 的資產授權。可決定安排哪些自己 BM 的員工管理對方 BM 的哪些共享資產。權限設定彈性高，不易發生整個 BM 被看光的問題。
  3. 系統工作人員 (System Users)：供內部程式開發人員使用。用於需要技術授權的任務，如設定轉換 API (Conversion API) 權杖。

廣告帳號與粉絲專頁權限管理:#

  • 廣告帳號權限 (Ad Account Permissions)：4 種:
    • 完整控制權 (管理員)：管理廣告、設定付款方式、編輯帳單。
    • 管理廣告活動 (標準權限)：操作廣告系統，但不含付款/帳單設定。
    • 查看成效 (唯讀權限)：只能查看報表。
    • 管理樣稿 (廣告小編)：上傳素材、設定貼文草稿，負責貼文編輯。
  • 粉絲專頁權限 (Fan Page Permissions)：5 種:
    • 管理員 (Admin)：最高權限，可對粉絲專頁做任何事，收所有通知，管理授權。
    • 編輯發布 (Editor)：發文、回文。
    • 小編 (Moderator)：管理留言、傳訊息 (狹義)。
    • 廣告主 (Advertiser)：僅能以粉絲專頁身份投放廣告，無法進行日常管理 (發文、回文、傳訊)。代理商代操廣告理論上應要求廣告主權限即可。
    • 唯讀 (Analyst)：僅能查看。
  • 若代理商需要將廣告後台建立的貼文發佈到粉絲專頁上 (代 PO 文)，則需要客戶授予編輯權限。
  • Instagram 的管理流程與權限設定與粉絲專頁完全相同。
  • 其他 BM 資產 (Pixel, Catalog) 的管理權限通常較簡單，取決於您的需求分配。

重要提醒：網域認證 (Domain Verification):#

  • 一個網域只能被一個企業管理平台認證。一旦認證完成，該網域即為該 BM 資產，無法被其他 BM 重複認證。
  • 若您的網站網域被現任代理商的 BM 認證，未來更換代理商時，必須請原代理商的 BM 將網域使用權分配給新的代理商 BM。每次更換廠商都需要重新分配。
  • 這被認為是 BM 設計上的瑕疵。切勿隨意讓廠商認證您的網域，最好由客戶自己的 BM 進行認證。
