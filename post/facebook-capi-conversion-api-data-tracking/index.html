Facebook 轉換 API (CAPI)：應對資料限制的追蹤解方

Facebook CAPI 轉換 API Cookie-less 第一方數據 轉換連結器
2025/05/27 · 1 min · 177 words · Fermi Logan | 建議修改

因應 iOS 14.5 和 Cookie 限制導致的資料收集困難，Facebook 轉換 API (CAPI) 應運而生，其核心原理是避開瀏覽器，透過網站伺服器直接向 Facebook 伺服器傳輸再行銷與轉換資料，大幅提升數據精確度達 20% 以上；品牌可透過直接代碼安裝、GTM 等第三方合作夥伴或雲端 API 通道來安裝，以確保第一方數據的累積，而 Google Ads 也有對應的轉換連結器工具來應對此挑戰。

  • Facebook 轉換 API (CAPI) 的目的與背景：

    • CAPI 是 Facebook 廣告的追蹤工具. 用來收集網站再行銷資料和轉換資料.
    • 開發 CAPI 的原因：近年資料收集變得困難.
      • 瀏覽器因 GDPR 等資料保護政策限制個資收集.
      • iOS 14.5 更新嚴重限制廣告主透過行動裝置收集使用者資料，使用者可自由選擇是否分享.
      • 導致 Facebook 透過瀏覽器收集的資料越來越不可靠.
      • 瀏覽器限制第三方 Cookie (Google Chrome 已禁用).
      • Ad block (擋廣告工具) 也會影響廣告代碼的資料收集.
    • 傳統追蹤工具 (如 Facebook Pixel, Google/LINE 代碼) 都是透過瀏覽器的 Cookie 獲取資料. 資料透過瀏覽器回傳給媒體系統.
    • CAPI 的核心功能：避開瀏覽器的資料限制，直接從網站伺服器獲取資料並回傳給 Facebook 伺服器. 解決資料遺失或不可靠的問題.

  • 轉換 API (CAPI) 的技術原理：

    • 在網站埋入一段代碼或憑證 (金鑰/權杖)，允許 Facebook 伺服器與網站伺服器溝通.
    • 這段金鑰會對每個造訪網站的使用者發送 Facebook ID (FB 可讀懂的 ID).
    • 資料和 ID 直接回傳給 Facebook 雲端資料庫，不透過瀏覽器保留或傳輸.
    • 網站完成網址驗證後，也默認 Facebook 可直接分享該網域的資料.
    • 技術上類似 Facebook 用伺服器資料與具有第一方 Cookie 使用權限的使用者 ID 同步.
    • 優勢：資料量體、事件和轉換的精確度可提升 20% 以上. 不再受瀏覽器連線問題、載入錯誤、Ad block 影響.

  • 安裝轉換 API 的方法：

    • 直接安裝代碼/權杖：最直接的方法.
    • 透過 Facebook 支援的第三方代碼合作夥伴：建議廣告主自己使用. Facebook 官方合作夥伴包括 Shopify, WooCommerce, Google 代碼管理工具 (GTM), Salesforce 等. 安裝流程中有引導，登錄第三方工具即可連結使用. 透過合作夥伴工具，默認 Facebook 有權存取網站伺服器資料. 這是少數 FB 與 Google 工具合作的機會.
    • 透過 DNS 伺服器的 API 通道：較技術性，直接透過主機伺服器進行. 需將 Facebook Pixel 連線雲端伺服器 (如 AWS)，提供網站位置和 DNS 登錄證明. 不需埋碼，但需要雲端伺服器，會產生額外費用 (如 AWS 約 80-100 美元/月).

  • Google Ads 的對應工具：轉換連結器 (Conversion Linker)

    • 用途與 Facebook 轉換 API 一樣：避開瀏覽器，直接從網站伺服器獲取資料.
    • 安裝更簡單：若使用 GTM，只需在 GTM 中新增「轉換連結器」代碼，設定為全站導入 (all page) 的觸發條件即可.

  • Cookie-less 時代與數據收集的重要性：

    • 媒體追蹤與資料收集受很大影響.
    • 部分電商客戶在 iOS 14.5 後商城資料收集量變少，影響再行銷和轉換廣告，難以使用 CPA 出價.
    • 盡早收集第一方資料非常重要 (如會員系統、CRM 資料，記錄使用者與品牌接觸點). 最好整合網站使用行為數據.
    • 行銷人員需趁早累積自己的行銷資產，否則獲客成本、媒體成本會越來越高. 需要居安思危，提早準備.
